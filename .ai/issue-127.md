# Add .env parser and validator utility with CLI

Добавить модуль src/envcheck.py.

Нужно реализовать парсер и валидатор .env-подобных файлов.

Формат входного файла:
- строки вида KEY=VALUE
- пустые строки игнорируются
- строки, начинающиеся с #, считаются комментариями и игнорируются
- пробелы вокруг KEY и VALUE нужно обрезать
- если строка не содержит символ =, она считается невалидной
- если KEY пустой после trim, строка считается невалидной
- если один и тот же KEY встречается несколько раз, использовать последнее значение

Реализовать функции:

1. parse_env(path: str) -> tuple[dict[str, str], list[str]]
   Возвращает:
   - словарь переменных
   - список ошибок парсинга

   Ошибки парсинга должны быть строками строго такого вида:
   - line <N>: missing '='
   - line <N>: empty key

   Где <N> — номер строки в файле, начиная с 1.

   Требования:
   - пустые строки и комментарии не считаются ошибками
   - строки без = добавляются в ошибки как missing '='
   - строки с пустым ключом добавляются в ошибки как empty key
   - при повторяющихся ключах используется последнее значение

2. validate_env(data: dict[str, str]) -> list[str]
   Проверки:
   - если есть PORT, он должен быть целым числом от 1 до 65535
   - если есть DEBUG, он должен быть только true или false
   - если есть HOST, он не должен быть пустым
   - если есть TIMEOUT, он должен быть целым числом >= 0

   Ошибки валидации должны быть строго:
   - PORT: must be an integer
   - PORT: must be between 1 and 65535
   - DEBUG: must be true or false
   - HOST: must not be empty
   - TIMEOUT: must be an integer
   - TIMEOUT: must be >= 0

3. check_env(path: str) -> tuple[dict[str, str], list[str]]
   - вызывает parse_env
   - затем validate_env
   - возвращает итоговый dict и объединённый список ошибок
   - ошибки парсинга должны идти раньше ошибок валидации

CLI:
python -m src.envcheck --file .env

Поведение CLI:
- если ошибок нет:
  - печатает переменные в stdout
  - ключи должны быть отсортированы по алфавиту
  - формат каждой строки строго:
    KEY=VALUE
  - exit code 0

- если ошибки есть:
  - печатает ошибки в stderr
  - каждая ошибка на новой строке
  - exit code 1
  - stdout должен быть пустым
